The Technology

DS-STAR: an iterative, self-correcting agent pipeline

Error recovery
Output
One step at a time
Planner generates one plan step per round, conditioned on the previous execution result. Mirrors how a data scientist explores data iteratively.
Backtracking Router
If a plan step is wrong, Router identifies which one and truncates the plan. Planner regenerates from that point — not just appending a new step.
LLM-as-judge Verifier
Verifier checks if the result actually answers the original query — not just whether the code ran. Drives iteration or terminates the pipeline.
Isolated Docker sandbox
Every generated Python script runs in a container — no network, read-only data mount, 2 GB memory cap. Data never leaves CBUAE infrastructure.

Platform Modes

Two modes for every supervisory workflow

FIP-Insight

Specific Analysis

For targeted questions that need a precise, verified, data-backed answer. DS-STAR iterates through files to produce a result with charts, tables, and a plain-English narrative.

"Which LFIs have the highest card fraud loss rate relative to transaction volume in Q1 2025, and how does each compare to the peer median?"
Returns tables, charts, and plain-English narrative
Self-corrects via Verifier + Router loop (up to 20 rounds)
Typically 3–5 rounds · under 30 seconds end-to-end
Export to Excel or PDF · full audit trail
FIP-Research

Thematic Reports

For open-ended research queries. Decomposes the question into sub-questions, runs DS-STAR on each, then synthesises a leadership-ready report where every claim is cited.

"Generate a thematic analysis of APP fraud growth across UAE banks in 2024 — include typology breakdown, channel analysis, YoY trends, and LFI outliers."
Structured report with sections, tables, and findings
Every claim cited to the sub-question and code that proved it
Evaluator refinement loop checks completeness before delivery
Export to Word or PDF · examination-ready evidence package

CBUAE-Native Intelligence

DS-STAR enhanced for UAE financial supervision

Query Clarity Agent
Validates queries before planning. Maps business language to data columns ("high-risk customers" → risk_band = 'High'). Flags ambiguous queries upfront.
Domain-Aware Analyzer
Recognises CBUAE schemas — LFI fraud reports, SAR/STR filings, KYC data, complaint registers — without being told. Produces richer data descriptions per department.
Regulatory Knowledge Injection
UAE AML law, FATF 40 Recommendations, CTR threshold (AED 40,000), fraud typology definitions — injected into Planner and Coder prompts. Correct analysis by default.
Plain English Explainer
Converts DataFrames, statistics, and charts into plain-language narratives tailored to each department's regulatory role and terminology — no data literacy needed.
PII Masking Layer
Account numbers, national IDs, and names are automatically masked before delivery. Raw PII accessible only to authorised roles. Masking happens before the analyst sees output.
Immutable Audit Log
Every query, plan step, code version, execution result, and output is timestamped and logged. Tamper-evident, user-attributed, and examination-ready.
