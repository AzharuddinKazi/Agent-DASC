The Technology
DS-STAR: 8-agent self-correcting pipeline achieves +32pp accuracy lift on hard tasks through iterative reasoning
Key Finding A single LLM call achieves 12.7% on hard analytical tasks. DS-STAR's iterative pipeline — with backtracking, verification, and up to 20 reasoning rounds — achieves 45.2% (+32pp). The Analyzer alone accounts for −18pp of accuracy if removed.

Agent Roles & Mechanism
Analyzer
Profiles every file before planning begins. Detects CSV/JSON/Excel/unstructured. Produces D: per-file description with columns, dtypes, null rates, sample rows. Critical: ablation −18pp without it.
Planner
Generates exactly one plan step pₖ per round, conditioned on (q, D, p₀..pₖ₋₁, rₖ₋₁). Mirrors iterative notebook exploration. Accumulates plan across rounds.
Coder
Extends prior script sₖ₋₁ with new plan step. Uses absolute paths, memory-safe row limits, multi-delimiter CSV detection. Pure Python only.
Executor
Runs sₖ in isolated Docker container. Returns stdout rₖ, stderr, exit code. Per-execution timeout. Output capped to prevent context overflow.
Debugger
Activated only on exit_code ≠ 0. Receives (sₖ, stderr, traceback). Fixes code errors only — never creates new plan steps. Max 3 attempts per round.
Verifier
LLM-as-judge. Receives full context (q, p₀..pₖ, sₖ, rₖ). Returns "sufficient" or "insufficient". Checks semantic answer quality, not just execution success.
Router
On "insufficient": returns "Add Step" (append) or index l (truncate plan to p₀..pₗ₋₁). Truncate-and-regenerate — not direct correction — exploits LLM sampling diversity.
Finalizer
Receives (sₖ, rₖ, formatting_guidelines). Generates short formatting script if needed. Applies rounding, chart formatting, output structure. Delivers clean final answer.
Critical: Removing the Analyzer alone drops hard-task accuracy 45.2% → 26.98% (−18pp). It is the single most impactful component in the entire pipeline.

Platform Modes
FIP-Insight for specific queries in seconds; FIP-Research for leadership-ready thematic reports
Key Finding One platform, two operating modes. FIP-Insight runs the 8-agent DS-STAR core for targeted factoid queries. FIP-Research wraps DS-STAR × n sub-questions with a Report Writer and Evaluator for open-ended analysis — every claim in the report is cited to code.

CBUAE-Native Intelligence
9 purpose-built enhancements embed UAE regulatory expertise directly into the DS-STAR pipeline
Key Finding DS-STAR out-of-the-box has no domain knowledge. The CBUAE layer adds schema awareness, UAE law (Decretal Federal Law 20/2018, FATF 40), PII protection, and examination-ready audit logging — turning a research paper into a production supervisory tool.
