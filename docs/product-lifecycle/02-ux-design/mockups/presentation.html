Section 03 · Technology — DS-STAR Architecture
CONFIDENTIALJune 2026
DS-STAR Architecture
8-agent self-correcting pipeline — iterative, verifiable, backtracking-capable
Key Finding A single LLM call achieves 12.7% on hard analytical tasks. DS-STAR's iterative loop reaches 45.2% (+32pp). Removing the Analyzer alone drops accuracy by 18pp — it is the single most critical component. The backtracking Router prevents the system from getting stuck on wrong plan steps.

Agent Roles
Analyzer
Profiles every file before planning. Auto-detects CSV/JSON/Excel/unstructured. Produces D: per-file description with columns, dtypes, null rates, sample rows. Critical: ablation −18pp without it.
Planner
Generates exactly one plan step pₖ per round, conditioned on (q, D, p₀..pₖ₋₁, rₖ₋₁). Mirrors iterative notebook exploration. Accumulates steps across rounds.
Coder
Extends prior script sₖ₋₁ with new plan step. Uses absolute paths, memory-safe row limits, multi-delimiter CSV detection. Pure Python only — no shell commands.
Executor
Runs sₖ in isolated Docker container. Returns stdout rₖ (capped 16KB), stderr, exit code. Per-execution timeout prevents infinite loops.
Debugger
Activated only on exit_code ≠ 0. Receives (sₖ, stderr, traceback). Fixes code — never creates new plan steps. Max 3 attempts per round before passing best-effort.
Verifier
LLM-as-judge receives full context (q, p₀..pₖ, sₖ, rₖ). Returns "sufficient" or "insufficient". Checks semantic answer quality — not just execution success.
Router
On "insufficient": "Add Step" (append) or index l (truncate to p₀..pₗ₋₁). Truncate-and-regenerate exploits LLM sampling diversity — prevents local minima.
Finalizer
Receives (sₖ, rₖ, formatting_guidelines). Applies rounding, template, chart formatting. Invokes department output template by name if specified.
