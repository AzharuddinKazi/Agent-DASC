Solution · Two Modes
FIP-Insight answers specific questions; FIP-Research synthesises thematic reports — one platform covers all five CBUAE departments from day one
Query routing is automatic: the Query Clarity Agent classifies intent and directs to Mode A (DS-STAR) or Mode B (DS-STAR+) before any analysis begins.

DS-STAR · 8-agent self-correcting pipeline · single query

  • Chat-style response — a direct answer, not a document
  • Linear follow-up questions supported in same thread

All 8 agents in V1

Planner

›
Coder
Full script sₖ per round — not a delta. Q3 ✓

Executor
Docker sandbox. 1 container per task. Q1 ✓

Debugger
2-step: summarizer + fixer. Gets (s, tb, D). Max 3 attempts. Q15 ✓

Section 03 · Technology
CBUAE Intelligence Layer · 9 Enhancements
Five enhancements ship in V1; four are deferred to V2 to protect delivery scope and validate the core capability with analysts first
V2 items are architecturally accommodated via adapter patterns in V1 — deferral is scope discipline, not rework. All V1 items confirmed in PRD v1.0.
Ships in V1 ✓
E1 · Query Clarity AgentV1
Maps "high-risk customers" → risk_band='High'. Routes Mode A vs B. Shows clarification pop-up for ambiguous queries before pipeline starts.
E2 · Domain-Aware AnalyzerV1
Recognises CBUAE schemas: fraud loss reports (lfi_id, fraud_type, channel, amount_aed, detection_method), SAR/STR records, KYC/CDD, market surveillance.
E3 · Natural Language ExplainerV1
Post-Finalizer node. "ABC Bank card fraud rate is 3.2× the peer median at 0.18% of transaction volume." Non-technical analysts read this directly — no raw DataFrames.
E4 · LLM Provider AdapterV1
Single config flag: Gemini | Ollama | Azure OpenAI. Enables on-premise air-gapped deployment. Zero agent-level code changes when switching provider.
E5 · Immutable Audit LogV1
Append-only. Every query, plan step, code version, result linked by task_id. Forever retention. Admin: all records. Analyst: own queries only. Regulatory examination ready.
Deferred to V2 🔮
E6 · PII MaskingV2
V1 data is analyst-uploaded files, not live production DBs. Designed when live connectivity added in V2 — actual schemas unknown until then (Feature Discussion, Topic 7).
E7 · Regulatory Knowledge InjectionV2
Dept-specific domain knowledge in Planner/Coder prompts: AML/CFT law, FATF 40, CPR, penalty frameworks. V1 treats all analysts identically (Topic 6).
E8 · Supervisory Report TemplatesV2
Pre-built dept templates: Fraud Benchmarking, SAR Trend, Consumer Scorecard, Evidence Package. Analyst picks template vs free-form. Added once core system is proven.
E9 · Live Database ConnectivityV2
Informatica Data Catalog integration. Org-wide access reflected automatically. Adapter pattern accommodated in V1 for seamless plug-in when CBUAE IT is ready.
Why defer? — Scope discipline
V1 proves the core capability. Dept customisation adds complexity without validating the core hypothesis. V2 customises a system analysts already trust through daily use.

❌ Dropped or Changed vs Paper
Separate Evaluator Agent DROPPED
Generator handles both decomposition AND gap-finding (Q11). No separate node.
20-Round Default CHANGED
Reduced to 5–6. Paper's hard tasks avg 5.6 rounds. Extension UX handles edge cases.
Debugger Design CHANGED
1-step → 2-step (traceback summarizer + fixer). Gets (s, traceback, D) — D critical for semantic error fixes (Q15).
Finalizer Design CHANGED
Text-only → runs formatting script in Docker. Same execution model as Coder for consistency (Q16).
Docker Lifecycle CHANGED
Per-round → per-task. Container reused across all rounds. Lower overhead (Q1).
Delta/Patch Scripts DROPPED
Complete script sₖ every round — not incremental. Simpler, no patch-merging bugs (Q3).
