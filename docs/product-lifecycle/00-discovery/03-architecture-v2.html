🔧  12 Issues Corrected from Gemini Architecture v1

❌ #1  DS-STAR+ misrepresented
✅ Fixed: DS-STAR+ is a wrapper, not a peer agent
Report Writer was placed as a peer to Analyzer/Planner. DS-STAR+ is a separate orchestration layer that calls DS-STAR n times as its inner engine.
❌ #2  Sub-Question Generator missing
✅ Added: Core DS-STAR+ component
Decomposes open-ended queries into focused sub-questions. Without it, DS-STAR+ cannot function.
❌ #3  Evaluator missing
✅ Added: DS-STAR+ refinement loop
The Evaluator checks report quality and triggers additional sub-questions if the report is insufficiently deep. Paper cites positive correlation with refinement rounds.
❌ #4  LLM tiering: same model size
✅ Fixed: 11B (Flash) vs 70B (Pro)
Llama 3.1 70B vs Llama 3.3 70B is the same parameter count — no real cost/quality tradeoff. Corrected to Llama 3.2 11B (Flash) vs Llama 3.1 70B (Pro).
❌ #5  Executor shown twice
✅ Fixed: Executor orchestrates, doesn't run inside sandbox
Executor is the DS-STAR agent that spawns the container. It does not live inside the sandbox. The container is the ephemeral runtime.
❌ #6  Temporal + Celery both listed
✅ Decided: Celery + Redis only
Temporal and Celery are competing solutions. Celery+Redis is simpler, well-understood, and sufficient for this use case. Temporal adds operational complexity with marginal benefit on-premise.
❌ #7  Query Clarity Agent missing
✅ Added: Pre-Planner validation node
Validates answerability, maps domain terms to data columns, and routes to Mode A or Mode B. Critical for non-technical analyst experience.
❌ #8  Auth / RBAC entirely absent
✅ Added: Security & API Gateway Layer
Multi-department system at a financial regulator requires SSO authentication, department-scoped data access enforcement, and per-LFI permissions.
❌ #9  Audit log conflated with MinIO
✅ Fixed: Separate append-only audit store
Object storage is wrong for audit logs — they must be append-only, tamper-evident, and queryable for regulatory examination. MinIO is for reports/artifacts only.
❌ #10  NL Explainer missing
✅ Added: Output Pipeline layer
Non-technical analysts need plain English narratives, not raw DataFrames. Explainer converts technical results into department-tailored explanations before delivery.
❌ #11  PII Masker missing
✅ Added: Output Pipeline layer
Central bank handling customer financial data must mask account numbers, national IDs, and names before output delivery. Role-controlled unmasking for authorized investigators.
❌ #12  Observability & data ingestion missing
✅ Added: Observability + Data Ingestion layers
Production system requires Prometheus/Grafana for LLM cost tracking and error alerting. Ingestion pipeline explains how LFI data reaches the read-only volumes.
